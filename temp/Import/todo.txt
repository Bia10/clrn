1. Написать класс для отправки
2. Написать и отладить шедулер через ДБ
3. Отрефакторить
	
1. Исходные данные:
	- xml с импортируемыми данными
	- xml сформированный ядром из имеющихся данных(аналогичный экспортируемому из консоли)
	- xsl шаблон для формирования конечного xml документа, данные из которого будут импортироваться
2. При импорте консоль анализирует импортуремый xml и имеющиеся данные, формируя xsl шаблон.
3. Шаблон формируется на стороне консоли с учетом заренее известной структуры xml документа, который сгенерирует ядро.
4. Применяя преобразование исходного xml с импортируемыми данными с учетом xml сгенерированоого ядром можно 
	- разрешать конфликты для любой сущности или ссылки
	- изменять любые данные, находящиеся в БД, методом генерации данных для импорта.
5. Сложность в написании редактора для генерации xsl шаблона
